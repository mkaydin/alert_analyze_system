# T1047: Windows Management Instrumentation

## Description
Windows Management Instrumentation (WMI) is a Microsoft administrative framework for managing Windows systems and networks. WMI can be abused by adversaries for execution, discovery, lateral movement, and persistence. WMI is accessible via PowerShell, VBScript, wmic.exe, and the WMI API.

## Common Abuse Techniques
- **Process execution:** `wmic.exe process call create "cmd.exe /c whoami"`
- **Remote execution:** `wmic.exe /node:TARGET /user:DOMAIN\Admin process call create "payload.exe"`
- **Discovery:** `wmic.exe /node:TARGET os get name`, `wmic.exe qfe get`, `wmic.exe product get name`
- **Service manipulation:** `wmic.exe service where name='service' call change startmode auto`
- **Data collection:** `wmic.exe /node:TARGET path Win32_Process get *`

## Detection
- Event ID 4688 for wmic.exe process creation with suspicious command lines
- Sysmon Event 1 for wmic.exe execution
- Network connections from wmic.exe to remote systems via DCOM (port 135)
- WMI Activity log (Microsoft-Windows-WinRM/Operational)
- Event ID 5861 (WMI provider event) for persistent WMI subscriptions
- Look for wmic.exe with `/node:` parameter for remote execution

## WMI for Persistence (T1546.003)
Adversaries can use __EventFilter, __FilterToConsumerBinding, and CommandLineEventConsumer to create persistent WMI event subscriptions that execute code based on system events.

## MITRE Mapping
- **Tactic:** Execution
- **Platform:** Windows
- **Permissions Required:** Administrator (for remote)
