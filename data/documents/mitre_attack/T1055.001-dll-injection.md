# T1055.001: Process Injection — DLL Injection

## Description
Adversaries may inject dynamic-link libraries (DLLs) into processes to execute arbitrary code in the address space of another process. DLL injection involves writing a malicious DLL path into a target process and creating a remote thread to load it. This evades detection by running under the context of a legitimate process.

## Detection
- CreateRemoteThread, WriteProcessMemory, VirtualAllocEx API calls targeting other processes
- Suspicious DLLs loaded by legitimate processes (e.g., notepad.exe loading unknown DLL)
- Event ID 4688 showing processes created with `CREATE_SUSPENDED` flag
- Named pipe connections between suspicious processes

## Common APIs Used
- `OpenProcess` → `VirtualAllocEx` → `WriteProcessMemory` → `CreateRemoteThread`
- `QueueUserAPC` (APC injection variant)
- `SetWindowsHookEx` (SetWindowsHook injection)
- Process hollowing: unmapping legitimate code, writing malicious code, resuming thread

## MITRE Mapping
- **Tactic:** Execution, Defense Evasion, Privilege Escalation
- **Platform:** Windows
- **Permissions Required:** Administrator, SYSTEM
