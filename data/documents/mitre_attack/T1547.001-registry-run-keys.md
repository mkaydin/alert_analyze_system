# T1547.001: Boot or Logon Autostart Execution — Registry Run Keys

## Description
Adversaries may add entries to the Windows Registry "Run" keys to achieve persistence. These keys cause programs to execute automatically when a user logs on or the system boots. Common locations include `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` and `HKLM\...\Run`.

## Detection
- Monitor registry modifications to Run keys (Event ID 4657, 13)
- Look for new or modified values pointing to unusual file paths
- Non-Microsoft executables in Run keys, especially with suspicious names
- RunOnce keys (used once at next logon, then deleted)

## Registry Paths
- `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- `HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce`
- `HKLM\Software\Microsoft\Windows\CurrentVersion\Run`
- `HKLM\Software\Microsoft\Windows\CurrentVersion\RunOnce`
- `HKLM\Software\WOW6432Node\Microsoft\Windows\CurrentVersion\Run`

## Example
A registry Run key created under HKLM pointing to `C:\ProgramData\scheduler.exe` with auto-start at every boot as SYSTEM.

## MITRE Mapping
- **Tactic:** Persistence
- **Platform:** Windows
- **Permissions Required:** User, Administrator
