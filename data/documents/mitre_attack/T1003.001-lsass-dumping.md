# T1003.001: OS Credential Dumping — LSASS Memory

## Description
Adversaries may attempt to access credential material stored in the Local Security Authority Subsystem Service (LSASS) process memory. LSASS stores credentials upon user logon that can be extracted with tools like Mimikatz, procdump, or Task Manager. Successful extraction provides plaintext passwords, NTLM hashes, and Kerberos tickets.

## Detection
- Process access to lsass.exe with suspicious access rights (PROCESS_VM_READ, PROCESS_ALL_ACCESS)
- Event ID 4656 (Handle to object requested) for lsass.exe
- Non-standard tools reading lsass dump files (.dmp created by procdump)
- `comsvcs.dll` usage via `rundll32.exe C:\Windows\System32\comsvcs.dll, MiniDump`
- Detection of Mimikatz or Mimikatz-like behavior in process memory

## Common Tools
- Mimikatz (`sekurlsa::logonpasswords`)
- ProcDump (`procdump.exe -ma lsass.exe lsass.dmp`)
- Task Manager (right-click lsass.exe -> Create dump file)
- `rundll32.exe comsvcs.dll, MiniDump <PID> lsass.dmp full`

## MITRE Mapping
- **Tactic:** Credential Access
- **Platform:** Windows
- **Permissions Required:** Administrator, SYSTEM
