# T1543.003: Create or Modify System Process — Windows Service

## Description
Adversaries may create or modify Windows services to execute malicious payloads repeatedly under a privileged context. Windows services are managed by the Service Control Manager (SCM) and can be configured to auto-start at boot, providing persistence.

## Detection
- Monitor service creation events (Event ID 4697, 7045)
- Look for services with unusual binary paths, especially in user-writable directories
- Services configured to run as SYSTEM or LocalSystem with non-Microsoft binaries

## Common Tools / Binaries
- `sc.exe create <service> binPath= "<malware.exe>"`
- `powershell New-Service -Name <service> -BinaryPathName <malware.exe>`
- `reg.exe` via `HKLM\System\CurrentControlSet\Services\<service>`
- Instersvr.exe, psservice.exe

## Example
A scheduler.exe binary installed as a service named "FortiClientUpdateSvc" with auto-start, running as SYSTEM, executing WinAPI calls via command line.

## MITRE Mapping
- **Tactic:** Persistence, Privilege Escalation
- **Platform:** Windows
- **Permissions Required:** Administrator, SYSTEM
